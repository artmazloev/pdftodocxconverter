## [Unreleased]

### Added
- Флаг **`--clean-svg-text`**: убирает обводку/ореол у SVG-текста (подписи на
  картах и т.п.). Обведённый текст Chrome печатает как Type3, и конвертеры бьют
  его на буквы пробелами («Г р у з и ю»); без обводки подписи становятся Type0 и
  извлекаются чисто. Цена — пропадает белый ореол под подписями.
- **Встроенный Manrope (OFL) + авто-инжект.** Документы, которые ссылаются на
  Manrope, но не встраивают его (грузят с Google Fonts по `<link>`), при рендере
  без сети теряли шрифт → дизайн в fallback. Теперь `html_prep` инжектит
  локальный статический Manrope (из `assets/`) для таких файлов и убирает
  Google-Fonts ссылки → рендер **offline и детерминированный**. Шрифт лежит в
  `pdf2docx_converter/assets/` под OFL.
- Инстансы вариативных шрифтов теперь получают **корректные имена по весам**
  (`updateFontNames`): `Manrope ExtraBold`, `Manrope SemiBold` и т.д. Это
  позволяет Adobe опознавать единое семейство и не подменять кириллицу/латиницу
  разными шрифтами внутри строки (раньше все веса звались одинаково).
- **HTML на входе**: `convert.py` принимает `.html` из `inputs/`. Внутри —
  `html_prep.py` заменяет вариативные `@font-face` на статические инстансы (фикс
  Type3), затем рендер через Chrome (`html_to_pdf.py`) и конвертация в DOCX.
  Решает проблему искажения шрифтов «на уровне нашего скрипта», без перегенерации
  исходников.
- Облачный движок **Adobe PDF Services** (`--engine adobe`) для дизайнерских PDF,
  с которыми не справляется локальный движок.
- **Постобработка `postprocess.py`**: чистка пробелов (всегда безопасно) и
  опциональная нормализация шрифтов. **Нормализация шрифтов меняет метрики и
  ломает спозиционированный макет Adobe**, поэтому она применяется только к
  выводу `--engine local` (потоковая вёрстка); для `--engine adobe` делается
  только чистка пробелов. Флаги `--no-postprocess`, `--font`,
  `--no-normalize-fonts`; запуск standalone на готовом DOCX.
- **`scripts/html_to_pdf.py`**: рендер HTML→PDF через headless Chrome (Playwright)
  с корректными print-ключами.
- **`scripts/make_static_fontface.py`**: превращает вариативный шрифт в
  статические `@font-face` (base64). Решает корень проблемы Type3: Chrome
  печатает variable-шрифты как Type3 (без имени) → конвертеры угадывают шрифт;
  статические инстансы дают Type0 и чистый текст. Доказано на Manrope.
- `requirements-html.txt` — опциональные `playwright` + `fonttools`.
- `requirements-adobe.txt` — опциональная зависимость `pdfservices-sdk`.
- `docs/ARCHITECTURE.md` — архитектура, сравнение движков, журнал решений.
- `CHANGELOG.md`.
- Документация по получению ключей Adobe и настройке `.env` (README).

### Changed
- README: вступление и «Как это работает» переписаны под два движка; обновлены
  структура проекта и раздел «Точность и ограничения».
- CONTRIBUTING: актуализированы структура, фазы и шаги тестирования.

### Known limitations
- Локальный движок (`pdf2docx`) плохо держит дизайнерскую вёрстку (векторные
  чарты, карточки) и **Type3-шрифты** (склеивает слова) — для таких файлов
  используйте `--engine adobe`.

### Added
- Базовый конвертер: батч `inputs/ → outputs/` (`convert.py`).
- Локальный движок на `pdf2docx` (`engine_local.py`).
- Pre-flight анализатор: цифровой PDF vs скан (`analyzer.py`).
- Инфраструктура: шаблоны issues/PR, CI (GitHub Actions), smoke-тест,
  README с развёртыванием, CONTRIBUTING.
- Backlog задач в issues (#2–#11) с группировкой по фазам.
