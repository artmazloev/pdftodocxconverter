# Архитектура

## Цель

Локально (на Mac, из Cursor) превращать PDF в **редактируемый** DOCX,
максимально сохраняя исходный вид. Папка `inputs/` → папка `outputs/`.

## Главный компромисс

PDF — формат с **абсолютным позиционированием** (у каждого глифа свои
координаты). DOCX — формат с **потоковой вёрсткой** (абзацы/таблицы перетекают).
Эти модели несовместимы по своей природе, поэтому:

> 100% визуальной точности **и** 100% редактируемости одновременно недостижимы
> ни одним инструментом. Чем сложнее дизайн исходника, тем сильнее компромисс.

Мы целимся в **гибрид**: текст обычным редактируемым потоком, абсолютные фреймы —
только для вёрстки, которую иначе не передать.

## Конвейер

```
  HTML ─▶ html_prep (variable→static шрифты) ─▶ Chrome (Page.pdf) ─┐
                                                                   ▼
            ┌─────────────┐                              ┌──────────────┐
  PDF  ───▶ │  analyzer   │  цифровой / скан? ───────────▶│    engine    │
            └─────────────┘                              │ local | adobe│
                                                         └──────┬───────┘
                                                                ▼
                                                  postprocess (шрифты/пробелы)
                                                                ▼
                                                       DOCX ─▶ outputs/
```

| Модуль | Ответственность |
|---|---|
| `convert.py` | CLI, батч `inputs/→outputs/`, выбор движка, маршрутизация HTML/PDF |
| `pdf2docx_converter/analyzer.py` | Pre-flight: цифровой PDF vs скан (PyMuPDF) |
| `pdf2docx_converter/engine_local.py` | Офлайн-движок на `pdf2docx` |
| `pdf2docx_converter/engine_adobe.py` | Облачный движок на Adobe PDF Services |
| `pdf2docx_converter/html_prep.py` | фикс шрифтов в HTML: variable→static, инжект встроенного Manrope, удаление Google-Fonts, опц. снятие обводки SVG-текста |
| `pdf2docx_converter/assets/` | встроенный Manrope (OFL) для инжекта |
| `pdf2docx_converter/postprocess.py` | Чистка пробелов; нормализация шрифтов (только local) |
| `pdf2docx_converter/config.py` | Папки I/O, параметры качества |
| `scripts/html_to_pdf.py` | HTML→PDF через headless Chrome (Playwright) |
| `scripts/make_static_fontface.py` | variable-шрифт → статический `@font-face` (CSS) |

## HTML на входе (рекомендуется для дизайнерских документов)

Если документ изначально сверстан как HTML, лучший путь — подавать **HTML**, а не
PDF. `convert.py` для `.html`/`.htm`:
1. `html_prep` чинит шрифты: (а) встроенные **вариативные** `@font-face` →
   статические инстансы по используемым весам (с корректными именами); (б) если
   шрифт (Manrope) только ссылается, но не встроен — **инжектит локальный Manrope**
   из `assets/` и убирает Google-Fonts ссылки; (в) по флагу `--clean-svg-text`
   снимает обводку с SVG-текста (иначе подписи карт становятся Type3);
2. `html_to_pdf` рендерит чистый PDF через headless Chrome (правильные
   print-ключи: `prefer_css_page_size`, `print_background`, нулевые поля,
   `media=print`);
3. PDF идёт в выбранный движок (рекомендуется `--engine adobe`).

Подробно про шрифты и причины искажений — [FONTS.md](FONTS.md).

## Движки

### `local` — pdf2docx (по умолчанию)

- Полностью офлайн, бесплатно, без лимитов.
- Реконструирует поток Word: абзацы, простые таблицы, картинки, базовые стили.
- **Сильные стороны:** текстовые документы (письма, договоры, отчёты).
- **Слабые стороны:** дизайнерская вёрстка (инфографика, чарты из векторных
  фигур), нестандартные шрифты. Подробнее — раздел «Ограничения».

### `adobe` — Adobe PDF Services (Export PDF)

- Облачный (файл загружается в Adobe), бесплатный тариф ~500 конвертаций/мес.
- Заметно лучше держит сложную вёрстку и Type3-шрифты.
- Опциональная зависимость `pdfservices-sdk` (`requirements-adobe.txt`).
- Ключи: env / `.env` / `pdfservices-api-credentials.json` (см. README).
- **Не использовать** для документов, которые не должны покидать машину.

## Ограничения и корневая причина искажений (важно)

На практике подтверждено на реальном дизайнерском документе (бизнес-инфографика
из HTML, шрифт Manrope):

1. **Векторная графика ≠ объекты Word.** Бар-чарты, карточки, иконки — это сотни
   векторных фигур. У Word нет нативного редактируемого эквивалента; локальный
   движок превращает их в кашу из картинок и вложенных таблиц. → `--engine adobe`.
2. **Корень искажения шрифтов и склеенных слов — Type3-шрифты в PDF.** Когда в
   PDF шрифт лежит как **Type3** (контуры глифов без имени), любой PDF→DOCX
   конвертер (включая Adobe) угадывает шрифт поглифно → патчворк шрифтов и
   потерянные пробелы.
3. **Откуда берётся Type3:** headless **Chrome `Page.pdf()` печатает вариативные
   (variable) шрифты как Type3.** Доказано: один и тот же Manrope как variable
   `@font-face` (`font-weight: 200 800`) → Type3; как статические инстансы по
   весам → Type0 (именованный subset) и чистый текст. **Решение встроено в
   конвейер** (`html_prep`): для HTML-входа вариативные шрифты автоматически
   переинстансятся в статические до рендера.
   - **Обведённый текст** (SVG `paint-order:stroke`) Chrome тоже печатает как
     Type3 → подписи карт бьются пробелами (`Г р у з и ю`). Лечится флагом
     `--clean-svg-text` (снимает обводку; цена — белый ореол под подписями).
4. **Шрифт может быть не встроен.** Часть документов ссылается на Manrope, но
   грузит его с Google Fonts по сети. Без сети рендер уходит в системный fallback
   → весь дизайн не тот. Конвертер **инжектит встроенный Manrope** (`assets/`, OFL)
   и убирает Google-ссылки → офлайн и детерминированно.

Вывод: HTML-исходник → `convert.py` (авто-фикс шрифтов) → `--engine adobe`. Для
готовых дизайнерских PDF — `--engine adobe`; для текстовых — `local`.
Подробный справочник по шрифтам — [FONTS.md](FONTS.md).

## Журнал ключевых решений

| Решение | Почему |
|---|---|
| Python + pdf2docx как база | Лучший баланс «редактируемо + похоже» в опенсорсе |
| Гибридный режим, а не «всё во фреймы» | Сохранить редактируемость |
| Батч `inputs/→outputs/` вместо GUI | Запуск из Cursor одним скриптом (запрос пользователя) |
| OCR отложен | Нужны только цифровые PDF на старте |
| Добавлен Adobe-движок | Локальный движок не тянет дизайнерскую вёрстку |
| HTML-вход + variable→static шрифты | Чинит корень Type3 «на уровне скрипта», без перегенерации исходников |
| Встроенный Manrope (OFL) + инжект | Документы грузят шрифт с Google Fonts; без сети дизайн ехал. Инжект → офлайн и детерминированно |
| `--clean-svg-text` опционально (не по умолчанию) | Снятие обводки меняет вид (убирает ореол) — это выбор пользователя |
| Постобработка шрифтов — только для `local` | Смена метрик ломает спозиционированный макет Adobe |
