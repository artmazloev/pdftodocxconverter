# Шрифты: причины искажений и как мы их чиним

Большинство проблем «шрифты искажаются / слова склеиваются / на карте лишние
пробелы» при PDF→DOCX сводятся к тому, **как шрифт лёг в PDF**. Здесь — что мы
выяснили на реальных документах (бизнес-инфографика на Manrope, экспорт из HTML)
и какие механизмы это лечат.

## Корневая причина: Type3-шрифты

Когда в PDF текст нарисован шрифтом типа **Type3** (контуры глифов как мини-программы,
без имени шрифта), любой PDF→DOCX конвертер (включая Adobe) **угадывает шрифт
поглифно**. Итог — патчворк шрифтов (одно слово в нескольких разных) и потерянные
пробелы (`выходнарынки`, `Спасибозаработу`).

Откуда берётся Type3 в нашем случае — три источника, все воспроизведены:

| Источник | Что происходит | Фикс |
|---|---|---|
| **Вариативный (variable) шрифт** | Chrome `Page.pdf()` печатает variable-шрифты как Type3 | `html_prep`: variable → статические инстансы по весам |
| **Обведённый SVG-текст** (`paint-order:stroke`) | Текст с обводкой Chrome печатает как Type3 → буквы бьются пробелами (`Г р у з и ю`) | флаг `--clean-svg-text`: убрать обводку |
| **Сжатие сторонним сервисом** | Некоторые «сжиматели» PDF переводят шрифты в Type3 | не сжимать перед конвертацией |

Проверка, есть ли Type3 в PDF:
```python
import fitz
d = fitz.open("file.pdf")
print(sorted({(f[3], f[2]) for p in d for f in p.get_fonts(full=True)}))
# ('', 'Type3') в списке → будут искажения
```

## Доказательство (variable → Type3)

Один и тот же Manrope через Chrome `Page.pdf()`:
- `@font-face` с `font-weight: 200 800` (**variable**) → `('', 'Type3')` ❌
- статические инстансы по весам → `Manrope-ExtraBold/SemiBold/...` (**Type0**) ✅, текст чистый.

Метрики статического инстанса = метрики того же веса variable-шрифта, поэтому
**вёрстка при этом не едет**.

## Что делает конвертер автоматически

Для HTML-входа (`convert.py file.html`):

1. **variable → static.** Встроенные вариативные `@font-face` переинстансятся в
   статические по используемым весам, с корректными именами (`updateFontNames`),
   чтобы Adobe опознавал единое семейство и не дробил кириллицу/латиницу.
2. **Инжект встроенного Manrope.** Если документ ссылается на Manrope, но **не
   встраивает** его (грузит с Google Fonts по `<link>` или ждёт установленным),
   конвертер подставляет локальный Manrope из `pdf2docx_converter/assets/`
   (лицензия OFL) и убирает Google-ссылки → рендер офлайновый и детерминированный.

## Что делается по флагу

- **`--clean-svg-text`** — убирает обводку у SVG-текста (подписи карт и т.п.),
  чтобы они не становились Type3. Цена — пропадает белый ореол под подписями.

## Постобработка DOCX (опционально)

После конвертации (`postprocess.py`):
- **чистка пробелов** — безопасно всегда (схлопывает множественные пробелы);
- **нормализация шрифтов** — сводит «угаданные» шрифты к одному. Применяется
  автоматически **только для `--engine local`**: смена метрик ломает
  спозиционированный макет Adobe, поэтому для Adobe делается только чистка пробелов.

Ручной ремап на готовом DOCX:
```bash
python -m pdf2docx_converter.postprocess file.docx --font Manrope --no-merge-runs
```

## Известные пределы

- **Одиночные пробелы внутри слов** (`предел ах`) автоматически не убираются —
  без словаря риск склеить реально разные слова (issue #13).
- Если в системе/у конвертера нет нужного шрифта, он подменяется. Для гарантии
  установи шрифт (напр. Manrope) локально или встрой в HTML.

## Чужие (не Manrope) шрифты

Авто-инжект сейчас знает только Manrope (он в проекте). Для другого шрифта:
встрой его в HTML как статический `@font-face` (можно через
`scripts/make_static_fontface.py`) или установи в систему перед рендером.
